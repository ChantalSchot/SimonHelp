Feeding schedule based on nutrients

Principle

In each feeding schedule, a maximum of 6 feeds can be used simultaneously.
Therefore, it is possible to formulate four feeds, each containing only one nutrient, and vary nutrient input to Simon directly.
In order to run a simulation, Simon needs input for:
  • protein  --> specify nitrogen, protein source and amino acid composition
  • glucose --> specify lactose and/or starch
  • fat         --> specify fat
  • rest        --> specify ash to fill up dry matter (moisture won't work)

How to

Feeds

Formulate 4 feeds, each containing one nutrient:

Nitrogen
Enter 0 for all other nutrients; specify the protein sources used and adapt the amino acid composition if you like.
Set the digestibility to 100%.
If you specify a feed that contains only protein, please have a look at the protein to nitrogen ratio in the protein sources screen and choose a nitrogen content of 1000/protein factor (in g/kg).

Crude fat
Enter 0 for all other nutrients and set the fat digestibility to 100%.

Lactose
Enter 0 for all other nutrients and set the lactose digestibility to 100%.
As an alternative, you may use a similar approach for starch.

Rest
Enter a desired content for ash.
Ash will not be used as a nutrient in the model, and can thus be used to fill up to total dry matter intake.
Enter 0 for all other nutrients.
This value is needed to determine the dry matter content and is used by the model to calculate net endogenous protein losses in the terminal ileum.

Feeding Schedule

Make a feeding schedule that uses these 4 feeds.
Enter meaningful quantities for these nutrients and make sure the ratio between nutrients makes sense.
Specify protein intakes, not nitrogen.
The data with which Simon is calibrated and tested contained rates of live weight gain ranging from 400 to 1500 g/d.
If your nutrient inputs do not support these rates of gain, you may get strange outputs, and the program may even crash.
