How-to-use

Buttons

New: Define a new simulation.

Copy: Copy the selected feeding schedule.

Delete: Delete the selected feeding schedule.

Save: Save a newly defined or changed feeding schedule.

Restore: Undo changes since last save.

Start simulation(s): Start the simulations (if all required information has been filled in).

Fields

Name: Enter a name for the feeding schedule (max 15 characters).

Description: Enter a description for the feeding schedule.

Initial values:
Start weight (kg): Enter the starting weight of the simulated calf.
Days on solid feed: Specify the number of days the calf has theoretically been on solid feed at the start of simulation.
Quantity of solid feed (kg): Specify how much solid feed (in kg) the calf has received before the start of the simulation.
Simulation stops at:
End weight (kg): Enter the end weight (in kg); once the calf reaches this weight, the simulation ends.
Max. simulation time (day): Specify how many days the simulation will last at most, if the end weight is not reached.

Genotype: Specify the genotype for the simulation.

Schedule milk replacers, concentrates and forages: Specify which feeding schedules are applied to the simulation.

How-to: Creating and running simulations

In this window, you can define up to four simulations simultaneously. A simulation is characterized by a combination of genotype (Genotype window) and feeding schedules for milk replacer, concentrate and forage (Schedules window), the latter depending on defined feeds of each type (Feed window).

The simulations running simultaneously will cover the same time or live weight range (depending on your preference). The initial weight should exceed 40 kg, but the model is only evaluated in the live weight range of 80 to 240 kg. Once all required information is filled out, click ✔ Save and the simulation can be started; up to four specified simulations will run consecutively.

The results are shown in the Output window, but are not stored in the database permanently. Only basic information that is used in the simulation, i.e. genotypes, feed, feeding schedules and the simulated combination of the previous, are saved. If one of these (e.g. a genotype) has been deleted, re-running the simulation will result in an error message.
